# Contributing to SBFT

## Getting started
- Open an issue to propose significant changes before coding.
- Use feature branches and keep pull requests focused.
- Run `python -m compileall sbft.py` and applicable tests before submitting.

## Coding guidelines
- Follow PEP 8 style; keep functions small and purposeful.
- Add docstrings/comments where clarity is needed.

## Testing
- If adding functionality, include minimal tests where feasible.
- Ensure dependencies are listed in `requirements.txt`.

## Security
- Do not disclose vulnerabilities publicly. Follow the process in `SECURITY.md`.
